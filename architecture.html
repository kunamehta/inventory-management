Filter Data Flow

1
User changes a filter
FilterBar.vue updates a shared ref in the useFilters() composable — one of selectedPeriod, selectedLocation, selectedCategory, or selectedStatus.
2
View detects the change
Each view (e.g. Inventory.vue) has a watch() on the relevant filter refs. Any change triggers loadData().
3
API call constructed in api.js
getCurrentFilters() builds the params object. api.js serializes non-all values into URL query params: ?warehouse=Tokyo&category=Sensors
4
FastAPI processes the request
apply_filters(items, warehouse, category, status) and filter_by_month(items, month) narrow the in-memory list. Quarters like Q1-2025 are expanded to month ranges.
5
Pydantic validates & serializes
All responses pass through Pydantic response_model annotations (e.g. List[InventoryItem]), ensuring type-safe JSON output.
6
Vue reactivity re-renders the view
The response is assigned to a ref (e.g. items.value). Computed properties and the template automatically re-render with the filtered data. Detail rows open in modals, not new pages.
